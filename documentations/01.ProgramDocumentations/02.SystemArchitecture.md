# Rendszerterv

### 1. Rendszer architektúra

A rendszer egy többrétegű, konténerizált webalkalmazás:

- **Frontend:**  
    - React, Typescript, Tailwind CSS.
    - Single Page Application (SPA) felépítés, modern és reszponzív UI.
- **Backend:**  
    - PHP 8+, Laravel keretrendszer (11+).
    - Teljes REST API, amely a frontend összes adatát biztosítja.
- **Adatbázis:**  
    - MariaDB/SQLite, relációs adatmodell.
- **Deployment:**  
    - Docker Compose alapú virtualizáció, automatizált indulási/leállítási folyamatok.
- **Verziókezelés & Dokumentáció:**  
    - Verziók követése a GitHub-on: [Quotiva repozitórium](https://github.com/NemethTamas1/QuotivaProject)
    - Minden fontos dokumentáció markdown-ban készül, hogy hosszú távon is jól karbantartható legyen.

### 4. Technológiai stack

- **Frontend:** React Typescript, Next.js, Tailwind CSS (Composition API, SPA logika)
- **Backend:** PHP 8+, Laravel PHP(REST API, validáció, hitelesítés)
- **Adatbázis:** SQLite
- **Konténerizáció:** Docker, Docker Compose
- **Tesztelés:** Laravel beépített tesztelési keretrendszer, SeleniumIDE, Playwright
- **Dokumentáció:** Markdown

### 6. Biztonsági alapszabályok

### 8. Deployment és üzemeltetés

- **Indítás:**  
    - A rendszer Docker Compose-al könnyedén indítható fejlesztői üzemmódban(`docker compose -p quotiva-dev -f docker-compose-dev.yaml up -d`), leállítható, vagy akár az adatbázis is törölhető (`docker compose down -v`).  
    - Részletes indítási útmutató megtalálható a [README.md](https://github.com/NemethTamas1/QuotivaProject) fájlban.
- **.env kezelés:**  
    - Backend környezeti változók backend oldalon, `.env` fájlban.
    - Frontend környezeti változók frontend oldalon, `.env` fájlban.
- **Backup és logolás:**  
    - Ajánlott rendszeres adatmentés beállítása, naplófájlok rotálása.
